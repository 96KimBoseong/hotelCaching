- 프로젝트 계기 :  caching과 redis를 테스트 하기 위한 가장 적합한 주제라 생각하여 진행하게 되었습니다.
- 검색 기능에 caching 적용 이유 : 호텔 검색 후 받는 리스트들은, 호텔이라는 업종 특성상 자주 변하지 않는다 생각 했습니다. 그 리스트들을 캐싱해서 제공하면 높은 성능을 기대할수 있을거라 생각하여 도입 하였습니다.
